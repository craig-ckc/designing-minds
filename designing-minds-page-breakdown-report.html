Home Page

https://designingminds.co.za/

The home page introduces printable CAPS-aligned resources for Grades 3 to 7, plan options, grade/resource navigation, featured tests, testimonials, and newsletter capture.

200

Measured Signals

  • HTML payload: about 222 KB.
  • Stylesheets: 93.
  • Scripts: 105.
  • Images: 11.
  • Forms: 2.
  • Detected h1: 0.

SEO / Structure

  • Title: Designing Minds - Caps-Aligned Assessments.
  • Meta description missing.
  • No detected h1; first extracted heading is an h3.
  • Needs Organization, WebSite, and product/category structured data entry points.

Content / UX

  • Offer is understandable: CAPS-aligned resources for Grades 3 to 7.
  • Grade/category navigation is useful and should be preserved.
  • Plan naming should be tied clearly to term/year access and download rights.
  • CTA hierarchy should favor shopping by grade or bundle.
High

Main Issues

  • Too much builder/plugin code for a marketing entry page.
  • No proper top-level heading harms accessibility and search hierarchy.
  • Add-to-cart query links appear in page source; robots blocks these, but the static rebuild should avoid crawlable cart mutation URLs.
Migration Notes

React Rebuild

  • Build the home page from content sections: hero, plan comparison, grade finder, featured products, testimonials, FAQ/contact prompt.
  • Use static product/category data to render grade cards and featured tests.
  • Pre-render the page and keep cart/payment code out of the initial home bundle.

200

  • Stylesheets: 89.
  • Scripts: 103.
  • Images: 19.

SEO / Structure

  • Meta description missing.

Content / UX

My Downloads

https://designingminds.co.za/my-account/downloads/

Post-purchase route where customers access purchased digital tests.

200

Measured Signals

  • Status: 200.
  • HTML payload: about 147 KB.
  • Current route depends on WooCommerce account/download functionality.

SEO / Structure

  • Account/download pages should generally be noindex.
  • Public navigation label “My Downloads” is clear.
  • Needs auth state handling in any non-WordPress rebuild.

Content / UX

  • This is critical for customer satisfaction after purchase.
  • Users need easy re-access to bought files.
  • Support path should be visible if a download fails.
Critical

Main Issues

  • This is the hardest migration area because static hosting alone does not provide authenticated downloads.
  • File protection, signed links, customer identity, and email fulfillment need a concrete design.
  • Legacy customers may need download migration or continued WordPress access.
Migration Notes

React Rebuild

  • Consider magic-link access rather than full accounts if the business model allows it.
  • Store files in protected object storage and issue expiring signed URLs.
  • Keep WooCommerce temporarily for historical downloads if migration risk is high.

Sitemap and Robots

/sitemap.xml and /robots.txt

Indexing control and crawler discovery files.

200

Sitemap Findings

  • /sitemap.xml redirects to /wp-sitemap.xml.
  • Includes posts, pages, testimonials, Elementor/ElementsKit content, products, WPR templates, ShopEngine templates, taxonomies, and users.
  • Builder/template and user sitemap entries are not ideal for a curated public ecommerce presence.

Robots Findings

  • Disallows WooCommerce logs and upload internals.
  • Disallows ?add-to-cart= URLs.
  • Disallows /wp-admin/ and allows admin-ajax.php.
  • Points crawlers to the WordPress sitemap.
Medium

Main Issues

  • Sitemap is generated by WordPress defaults rather than a business-focused SEO plan.
  • Internal builder/template content types should not be emphasized.
  • User archive sitemap can be undesirable on a small ecommerce site.
Migration Notes

React Rebuild

  • Generate a clean sitemap containing Home, About, Shop, product pages, category pages, Contact, Privacy, Terms, and any editorial pages.
  • Exclude cart, checkout, account, downloads, admin, and form endpoints.
  • Generate robots.txt as part of the Vercel static build.
